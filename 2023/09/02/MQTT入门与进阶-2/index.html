MQTT入门与进阶 - 2

发布和订阅

发布消息


Topic Name: UTF-8 String
QoS: Integer

QoS 0: 消息可能丢失
QoS 1: 消息不会丢失但可能重复
QoS 2: 消息不会丢失也不会重复

Payload: Binary

保留消息

正常转发的同时, 也会在本地保存副本, 当有新的客户端建立订阅时, 保留消息就会下发给客户端.

订阅消息


Packet ID

Subscription List

可以一次订阅多个报文

Topic Filter 可以使用主题通配符

但发布时不能使用主题通配符

发布QoS = 2的消息, 但接收端QoS = 1, QoS = 2的消息将会被降级

如果没有超过最大QoS, 则保持不变

使用相同主题过滤器的新订阅将覆盖旧订阅
消息将转发给所有匹配的订阅

订阅回复报文

Reason Codes

Granted Maximum QoS - 可能小于订阅时的Maximum QoS

取消订阅


想要取消订阅的Topic Filter-> 完全的文本匹配 -> 实际订阅的Topic Filter

主题

类型 UTF-8字符串
最大长度 65535字节
区分大小写

主题通配符

  • 单层通配符



    一个单层通配符不能匹配多个层级, 但可以通过多个通配符实现
  • 多层通配符
    #可以匹配任意数量个层级


    单层和多层可以搭配使用

    主题名 在发布时不支持主题通配符
    主题过滤器 在订阅和取消订阅时使用, 支持主题通配符
  • 特殊主题
    以$开头的主题, 由服务器使用, 客户端只能订阅, 不能发布.

    #并不包含$SYS主题, 需要明确指定$SYS/#

最佳实践

  • 不建议主题以/开头或结尾
  • 尽量仅使用ASCII字符
  • 不建议在主题使用空格
  • 不建议使用#订阅所有主题
  • 尽量使用简洁的主题
  • 在主题中包含标识信息
